## What was actually broken (and what was fixed)

### 1. KOSPI ticker showing "---"
**Diagnosis:** Not a real bug. `/api/kospi` returns valid data (current: 7493.18, verified by curl). The `---` is what server-rendered HTML and no-JS clients (including Codex's web search and WebFetch) see, because the fetch only runs client-side in `useEffect`. Real browsers replace it with the live value within ~100ms of hydration.
**Fix shipped:** Seeded `useState` for `kospi` and `liveRate` from `currentSnapshot` so SSR + no-JS render real numbers, not `---`. The useEffect still overwrites with live data once JS hydrates. Silent `.catch(() => {})` calls replaced with `console.error` so future fetch failures are visible.
**Files:** `src/app/page.tsx`

### 2. Gas-price fetch silently failing in CI
**Diagnosis:** `scripts/update-data.ts:fetchGasPrice` wrapped Opinet in `try { ... } catch {}` with no logging. When Isaac's private `OPINET_API_KEY` started returning empty `{"RESULT":{"OIL":[]}}` (likely expired or restricted), the script swallowed the empty result. Cron still committed USD/KRW updates so the failure went unnoticed.
**Fix shipped:** Logged every Opinet failure with reason. Added automatic fallback to public sample key `F231013281` (already used by `/api/gas-prices`). Gas now updates: jumped from stale ₩1907 to live ₩2011.
**Files:** `scripts/update-data.ts`
**Action item for Isaac:** Renew or replace the private OPINET_API_KEY in Vercel + GitHub Actions secrets so we stop relying on the public sample.

### 3. Newsletter route faking success when not configured
**Diagnosis:** `/api/newsletter` returned `{ success: true }` when `BREVO_API_KEY` was missing. Any signups since launch were silently black-holed.
**Fix shipped:** Now returns HTTP 503 with a user-visible "Newsletter is not yet available" error. Front-end already surfaces `data.error`, so users see the real state. Once `BREVO_API_KEY` is added in Vercel, no code change needed.
**Files:** `src/app/api/newsletter/route.ts`
**Action item for Isaac:** Add `BREVO_API_KEY` to Vercel env vars when ready to accept signups.

### 4. Update-data script not capturing KOSPI
**Fix shipped:** Added `fetchKospi()` using Yahoo Finance `^KS11`. KOSPI is now written to `currentSnapshot.kospi` (currently 7493.18) on every cron run.
**Files:** `scripts/update-data.ts`, `src/lib/data.ts`

### 5. Hardcoded "as of" dates scattered across the site
**Diagnosis:** Dashboard ("2026.03.10"), CompareTable ("2026.03"), PresidencyComparison ("2026.03"), and `generateMonthlyData` (history end = `2026-03-01`) all bake the audit date into source.
**Fix shipped:** Added `LAST_UPDATED` + `LAST_UPDATED_LABEL` exports in `src/lib/data.ts`. All four locations now read from there. The update-data script bumps `LAST_UPDATED` on every successful run, so the labels advance with the data.
**Files:** `src/lib/data.ts`, `src/app/dashboard/page.tsx`, `src/components/CompareTable.tsx`, `src/components/PresidencyComparison.tsx`, `scripts/update-data.ts`

### 6. Timeline range label "Jan 2024 – Mar 2026" hardcoded inline
**Fix shipped:** Extracted to `KOREA_TIMELINE_RANGE_LABEL` in `src/lib/editorial.ts`. Update there when you add new events past March 2026.
**Files:** `src/lib/editorial.ts`, `src/app/page.tsx`

### 7. /api/tweets returning empty when Apify dataset purged
**Diagnosis:** Codex flagged the in-memory cache being lost on cold starts. Actually fine: the Apify dataset reads use `next: { revalidate: 900 / 3600 }`, which is Vercel's persistent data cache. Cold starts hit cache, not Apify. The real gap: if Apify returned an empty array (not a throw), the route returned empty without checking the in-memory cache.
**Fix shipped:** When Apify returns no items but cache has data, return cache marked `cached: true`.
**Files:** `src/app/api/tweets/route.ts`
